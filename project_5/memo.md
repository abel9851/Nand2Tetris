### Hack시스템 아키텍처 개요
- 시스템 구성
	- 하드웨어는 크게 CPU와 메모리로 구성된다.
- 메모리 구조(하버드 아키텍처)
	- 물리적으로는 레지스터의 집합체이나, 논리적으로는 **데이터 메모리(RAM)** 와 **명령어 메모리(ROM)** 로 엄격히 분리되어 있다.
	- 데이터 메모리(RAM): 물리적으로 범용 RAM, 스크린(Screen), 키보드(Keyboard) 3개의 칩으로 구성된다.
		- 설계 의도
			- 범용 컴퓨터는 OS가 메모리 자원을 관리하지만, Hack은 학습용 미니멀 시스템이다. 따라서 복잡한 '메모리 보호(Memory Protection)' 로직을 소프트웨어로 구현하는 대신, 물리적으로 칩을 나누어 하드웨어 수준에서 주소 공간 격리를 실현했다.
- 명령어 메모리(ROM)
	- 범용 컴퓨터는 저장장치에서 RAM으러 프로그램을 로드(Load)하지만, Hack은 프로그램이 이미 ROM에 상주하는 것을 전제로 한다.
	- 왜 ROM인가?
		- CPU의 본질은 "메모리에 있는 이진 코드를 가져와 연산하는 것"이다. 로드 과정(I/O, File System)의 복잡성을 배제하고, CPU 본연의 기능 구현에 집중하기 위해 불변의 메모리인 ROM을 선택한 것이다. 이는 게임보이 같은 임베디드 기기의 팩(Cartridge)방식과 유사하다.
- CPU(Central Processing Unit)
	- 구성
		- ALU, A-레지스터, D-레지스터, PC(Program Counter)로 이루어져 있다.
	- 역할
		- 메모리에서 데이터를 읽어 연산하고, 다음 실행할 명령어의 주소를 결정한다.
- I/O 인터페이스(Memory Mapped I/O)
	- 키보드와 스크린은 **메모리 맵** 이라는 인터페이스를 통해 제어된다.
	- 핵심원리
		- CPU가 각 외부 장치의 물지적 구동 방식을 몰라도, 특정 메모리 주소에 값을 쓰고 읽는 것만으로 조작이 가능하게 하는 기법이다.
	- 키보드
		- 특정 주소(24576)를 읽으면 현재 눈린 키의 아스키(ASCII)코드를 즉시 얻을 수 있다.
	- 스크린
		- 메모리 주소 범위(16384~24575)의 각 비트가 화면의 픽셀과 1:1로 대응(Mapping)되어 있다. 이 영역에 값을 기록하는 것 만으로 화면을 흑백으로 제어한다.
